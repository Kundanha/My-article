Valid Anagram

Do strings anagram hain ya nahi ‚Äî 3 alag tarike se check karo! üî§‚ú®

üü¢ Easy Level

Problem Kya Hai?

Do strings s aur t diye gaye hain. Tumhe batana hai ki kya t string s ka anagram hai ya nahi.

Anagram Kya Hota Hai?

Anagram matlab ek word ke letters ko rearrange karke doosra word banana. Dono words mein same characters hone chahiye, same frequency ke saath!

üéØ Example: "listen" aur "silent" ‚Üí true (same letters, different order)
üéØ Example: "rat" aur "car" ‚Üí false (different letters)
üéØ Example: "anagram" aur "nagaram" ‚Üí true ‚úÖ
1

Sorting Approach

Intuition (Sochne Ka Tarika)

Agar do strings anagram hain, toh unme exactly same characters honge with same frequencies! üß©

Toh simple idea: Dono strings ko sort kar do! Agar anagram hain, toh sorting ke baad dono strings identical ho jayengi.

üí≠ Socho jaise dono strings ke letters ko alphabetically arrange kar rahe ho ‚Äî "listen" ‚Üí "eilnst" aur "silent" ‚Üí "eilnst" ‚Äî dono same! ‚ú®

Algorithm (Steps)

  1. Pehle lengths check karo ‚Äî agar different hain, toh directly false return
  2. Dono strings ko sort karo
  3. Sorted strings ko compare karo
  4. Agar equal hain ‚Üí true (anagram hai!)
  5. Warna ‚Üí false
Python 
class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        if len(s) != len(t):
            return False
        
        return sorted(s) == sorted(t)

Time & Space Complexity

‚è±Ô∏è Time Complexity
O(n log n)
n log n + m log m
üíæ Space Complexity
O(n + m)
or O(1) - sorting algo pe depend
üìù n = string s ki length, m = string t ki length. Sorting mein log factor aata hai, isliye linear se thoda slow hai.
2

Hash Map Approach

Intuition (Sochne Ka Tarika)

Sorting ki jagah, hum directly character frequencies count kar sakte hain! üìä

Do hash maps (dictionaries) banao ‚Äî ek string s ke liye, ek string t ke liye. Har character kitni baar aaya, wo count karo.

Agar dono maps exactly same hain, toh anagram hai! üéØ

üí≠ Jaise exam mein attendance le rahe ho ‚Äî dono classes ke students ki count karke match karo!

Algorithm (Steps)

  1. Length check karo ‚Äî different hain toh false
  2. Do empty dictionaries banao: countS aur countT
  3. Dono strings pe simultaneously loop lagao
  4. Har character ka count apne respective dictionary mein increment karo
  5. End mein dono dictionaries compare karo
  6. Same hain ‚Üí true, different ‚Üí false
Python 
class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        if len(s) != len(t):
            return False
        
        countS, countT = {}, {}
        
        for i in range(len(s)):
            # s[i] ka count badao
            countS[s[i]] = 1 + countS.get(s[i], 0)
            # t[i] ka count badao
            countT[t[i]] = 1 + countT.get(t[i], 0)
        
        return countS == countT

Time & Space Complexity

‚è±Ô∏è Time Complexity
O(n + m)
Linear scan!
üíæ Space Complexity
O(1)
Max 26 characters only
üöÄ Better than sorting! Time O(n) hai. Space technically O(1) hai kyunki lowercase English letters mein max 26 unique characters hi ho sakte hain!
3

Hash Table using Array üéØ

Intuition (Sochne Ka Tarika)

Problem mein guarantee hai ki sirf lowercase English letters honge (a-z). Toh kyun na ek fixed-size array of 26 use karein! üí°

Smart trick: String s ke liye count increment karo, string t ke liye count decrement karo.

Agar anagram hai, toh har increment ka matching decrement hoga ‚Äî end mein sab counts 0 ho jayenge! üéâ

üí≠ Socho jaise bank balance ‚Äî deposit (s ke characters) aur withdrawal (t ke characters). Agar sab match kare, toh balance zero hoga!

Algorithm (Steps)

  1. Length check ‚Äî different hain toh false
  2. 26 size ka array banao, sab values 0
  3. Dono strings pe loop lagao:
  4. s[i] ke liye: count[index]++
  5. t[i] ke liye: count[index]--
  6. Array scan karo ‚Äî koi bhi value non-zero? ‚Üí false
  7. Sab zero? ‚Üí true (anagram confirmed!)
Python 
class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        if len(s) != len(t):
            return False
        
        count = [0] * 26  # 26 letters ke liye
        
        for i in range(len(s)):
            # 'a' se distance nikalo (0-25)
            count[ord(s[i]) - ord('a')] += 1  # increment
            count[ord(t[i]) - ord('a')] -= 1  # decrement
        
        # Check: sab zero hona chahiye
        for val in count:
            if val != 0:
                return False
        return True

ord() Function Kya Karta Hai?

ord('a') gives 97, ord('b') gives 98, and so on...
ord('a') - ord('a') = 0 ‚Üí index 0
ord('b') - ord('a') = 1 ‚Üí index 1
ord('z') - ord('a') = 25 ‚Üí index 25
Isse har character ka unique index mil jata hai! üî¢

‚è±Ô∏è Time Complexity
O(n + m)
Super fast linear!
üíæ Space Complexity
O(1)
Fixed 26 size array
‚ö° Most Efficient! No hashing overhead, sirf simple array operations. Interview mein ye approach sabse impressive lagti hai!

Common Galtiyan (Pitfalls)

Length Check Bhool Jana

Agar do strings ki lengths different hain, toh wo kabhi anagram nahi ho sakte! Ye check nahi karne se unnecessary processing hoti hai.

Always first step: if len(s) != len(t): return False

üí° "anagram" (7 letters) aur "ana" (3 letters) ‚Äî lengths hi different hain, compare karne ka koi matlab nahi!

Case Sensitivity Issue

Is problem mein sirf lowercase letters hain, toh case sensitivity issue nahi hai. But agar problem mixed case allow kare, toh dhyan rakhna! üî§

Python ‚ùå Wrong (if mixed case) 
# "Listen" != "silent"
# (L vs l different!)
return sorted(s) == sorted(t)
Python ‚úÖ Correct (mixed case) 
# Pehle lowercase karo
s = s.lower()
t = t.lower()
return sorted(s) == sorted(t)
